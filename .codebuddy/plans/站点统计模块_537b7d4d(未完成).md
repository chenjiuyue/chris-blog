## 产品概述

基于现有博客系统，扩展站点统计模块，提供更全面的数据可视化和分析功能。

## 现有功能

- **基础统计展示**：侧边栏显示文章数、浏览量、评论数、点赞数
- **访问日志记录**：`blog-recordVisit` 云函数记录访问到 `blog_visits` 集合
- **热门文章**：按浏览量排序的热门文章组件

## 新增功能

### 一、独立统计页面

- 创建 `/stats` 路由页面
- 数据可视化看板布局
- 响应式设计适配移动端

### 二、访问趋势分析

- 近 7/30 天访问量趋势图
- 日/周/月访问量统计
- 访问量峰值标记

### 三、内容统计分析

- 文章浏览量排行 Top 10
- 分类文章数量分布
- 标签使用热度
- 评论活跃度统计

### 四、访客分析

- 独立访客数（UV）统计
- 页面访问量（PV）统计
- 访问来源分析

### 五、数据聚合云函数

- 定时聚合每日统计数据
- 计算访问趋势
- 缓存统计结果提升性能

## 技术栈

基于现有技术栈扩展：

- **前端框架**: React 18 + TypeScript
- **图表库**: Chart.js 或 Recharts（轻量级，与 React 集成良好）
- **构建工具**: Vite 5
- **样式方案**: Tailwind CSS 3.4
- **后端服务**: CloudBase（NoSQL + 云函数）

## 技术方案

### 1. 数据库设计

#### 新增集合：`blog_daily_stats`（每日统计汇总）

```typescript
interface DailyStats {
  _id: string;
  date: string; // YYYY-MM-DD
  pv: number; // 页面访问量
  uv: number; // 独立访客数
  newPosts: number; // 新增文章数
  newComments: number; // 新增评论数
  newLikes: number; // 新增点赞数
  topPosts: { id: string; title: string; views: number }[];
  createdAt: string;
}
```

#### 扩展现有：`blog_visits` 访问日志

已有字段：postId, page, referrer, userAgent, ip, timestamp, date

### 2. 云函数设计

| 云函数名 | 功能 | 触发方式 |
| --- | --- | --- |
| `blog-aggregateStats` | 聚合每日统计数据 | 定时触发（每日凌晨） |
| `blog-getStatsOverview` | 获取统计概览数据 | 前端调用 |
| `blog-getVisitTrend` | 获取访问趋势数据 | 前端调用 |


### 3. 前端组件结构

```
src/pages/StatsPage.tsx          # 统计页面
src/components/stats/
  ├── VisitTrendChart.tsx        # 访问趋势图表
  ├── TopPostsChart.tsx          # 热门文章排行
  ├── CategoryDistribution.tsx   # 分类分布图
  ├── TagCloudStats.tsx          # 标签统计
  └── StatsCard.tsx              # 统计卡片组件
```

### 4. 图表库选择

推荐使用 **Recharts**：

- 基于 React 构建，组件化 API
- 轻量级，打包体积小
- 响应式支持良好
- TypeScript 支持完善

## 性能优化

1. **数据聚合缓存**：每日统计数据由云函数预计算并存储，避免实时聚合
2. **前端缓存**：统计数据缓存到 localStorage，设置合理过期时间
3. **按需加载**：图表组件懒加载，减少首屏加载时间
4. **分页查询**：大数据量时分页获取，避免一次性加载过多数据

## 设计风格

延续博客现有的 Editorial 风格，采用简约、专业的数据可视化设计。

## 页面布局

- 顶部：统计概览卡片（总访问量、文章数、评论数、点赞数）
- 中部左侧：访问趋势折线图
- 中部右侧：分类分布饼图
- 下部：热门文章排行榜、标签云统计

## 设计要点

- 使用柔和的渐变色彩区分数据类别
- 图表配色与博客主题色（#D4A574）协调
- 暗色模式下图表颜色自动适配
- 数据卡片使用圆角、阴影增强层次感
